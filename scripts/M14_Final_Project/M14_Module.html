REVIEW -- First Draft

Let me share some advice from watching hundreds of student projects.

First, scope ruthlessly. The most common mistake is building too much. A polished, working demo of one thing beats a broken prototype of five things. Every time.

Second, pick a problem you care about. If you are in supply chain, build a supply chain tool. If you are in healthcare, build a healthcare tool. Your domain expertise combined with AI skills is what makes the project unique.

Third, architecture matters. Think about your system design before you write a single line of code. Which LLM? Which tools? How do components connect? A clean architecture is easier to build, easier to debug, and easier to present.

Fourth, demo over slides. When you present, show the working system. Click through it live. Nothing is more convincing than a system that works in front of the audience.

Fifth, document everything. Your report should explain not just what you built, but why you made each technical decision. That is what separates an engineering project from a coding exercise.

Project Requirements

Your final project is a working AI system that demonstrates genuine engineering skill. Here is what is required.

Scope

Build a system that uses generative AI to solve a real problem. It must go beyond a simple API call -- your project should demonstrate at least three techniques from the course (e.g., structured output + RAG + agents, or prompt engineering + MCP + workflow automation).

Team Size

Teams of 1-3 students. Larger teams are expected to deliver proportionally more complex projects.

Project Ideas

These are suggestions to spark your thinking. You are welcome to propose your own idea.

Supply Chain Optimizer

Multi-agent system that monitors inventory levels, predicts demand, and generates reorder recommendations. Use LangGraph + structured output + RAG on historical data.

AI Teaching Assistant

Chatbot grounded in course materials via RAG. Answers student questions, generates practice problems, and explains concepts at different levels. Add MCP for document access.

Document Intelligence Pipeline

Upload contracts, invoices, or reports. AI extracts structured data, identifies risks, and generates summaries. Use vision + structured output + n8n automation.

Customer Support Bot

Multi-agent support system: classifier agent routes queries, specialist agents handle different categories, escalation agent identifies complex cases. Built with CrewAI or Agent SDK.

Multi-Agent Research Tool

Give it a research question. One agent searches the web, another analyzes papers, another synthesizes findings. Final agent writes a structured research brief.

Personal Finance Analyzer

Connect to bank transaction data (mock or real). AI categorizes expenses, identifies patterns, generates budget recommendations, and creates visual reports.

Code Review Assistant

Analyze GitHub pull requests. AI identifies bugs, suggests improvements, checks for security issues, and generates review summaries. Use Claude Agent SDK + GitHub API.

Healthcare Triage System

Patient describes symptoms. AI asks follow-up questions, assesses urgency level, recommends next steps. Guardrails ensure safe, non-diagnostic responses.

Rubric & Grading

Your project is graded on five dimensions. Each is weighted equally.

DimensionWeightWhat We Look For
Technical Complexity20%Uses 3+ course techniques meaningfully. Architecture is well-designed. Handles edge cases.
Code Quality20%Clean, documented code. Clear README. Reproducible setup. No hardcoded secrets.
AI Integration Depth20%AI is central to the system (not bolted on). Prompt engineering is thoughtful. Output is validated.
Presentation20%Clear demo. Good pacing. Answered questions well. Live system worked.
Documentation20%Technical report explains architecture, decisions, and results. Honest about limitations.

Grade Breakdown

  • A (90-100): Excellent execution across all dimensions. Impressive demo. Could be a real product.
  • B (80-89): Solid project with good technical depth. Minor issues in presentation or documentation.
  • C (70-79): Working system but limited complexity or polish. Missing some deliverables.
  • Below 70: System does not work, or AI is not meaningfully integrated.

Past Project Examples

These examples illustrate what makes a strong final project. Use them as benchmarks, not templates.

Example 1: Supply Chain Intelligence Dashboard

Team of 2. Built a multi-agent supply chain optimizer using CrewAI. A demand forecasting agent analyzed historical data via RAG. A procurement agent generated reorder recommendations with structured output. A reporting agent produced daily dashboards. Frontend built with React. What made it excellent: Clear architecture diagram. Live demo with real data. Honest discussion of where the agents made mistakes.

Example 2: Legal Contract Analyzer

Solo project. Used vision API to process scanned contracts, structured output to extract clauses, and RAG to compare against a database of standard terms. n8n workflow automated the pipeline from email attachment to final report. What made it excellent: End-to-end automation. Professional-quality output. The student had legal domain expertise that showed in the prompt engineering.

Example 3: AI Study Companion

Team of 3. Built a RAG-powered study assistant that ingested lecture notes and textbook chapters. Used LangGraph for multi-step tutoring conversations. Added guardrails to prevent giving direct exam answers. Evaluation framework tracked answer quality. What made it excellent: Thoughtful guardrails design. Evaluation metrics presented in the demo. Students could not "trick" it into cheating.

Example 4: Real-Time News Analyst

Team of 2. Multi-agent system using Claude Agent SDK. A web search agent gathered breaking news. An analysis agent identified bias and verified claims against trusted sources via RAG. A summary agent produced balanced briefings. MCP connected to a custom data source. What made it excellent: Creative use of MCP. Bias detection was genuinely useful. Clean code with comprehensive documentation.
