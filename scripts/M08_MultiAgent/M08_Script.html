Video 2: "CrewAI: Building Agent Teams"

Voiceover + animation -- 10-12 minutes -- 5 segments
PART A The Story Behind CrewAI
Voiceover + Animation ~2 min

CrewAI was created in late 2023 by Joao Moura, a Brazilian-born engineer who noticed the same problem over and over -- developers were writing hundreds of lines of boilerplate just to get two agents to talk to each other. His idea: what if multi-agent systems worked the way real teams work? Define who is on the team, what each person does, and how they collaborate. The framework handles the rest.

He open-sourced it in early 2024 and it took off. By 2025, CrewAI had raised funding and become one of the most popular multi-agent frameworks in Python. What makes it special is the abstraction. You do not think in nodes and edges. You think in people and jobs.

Storyboard -- CrewAI Timeline
PART B The Three Building Blocks
Voiceover + Animation ~3 min

Everything in CrewAI comes down to three things. An agent has a role, a goal, and a backstory. A task is the assignment -- a description, an expected output, and one assigned agent.

The crew brings it together. You list your agents and tasks, pick the process -- sequential or hierarchical -- and hit run. CrewAI handles execution, handoffs, and context sharing.

PART C The Code
Voiceover + Code ~3 min

Here is what it looks like in code.

from crewai import Agent, Task, Crew

# Define agents
researcher = Agent(
    role="Senior Researcher",
    goal="Find the 3 most relevant papers",
    backstory="Expert at academic research"
)

writer = Agent(
    role="Technical Writer",
    goal="Write a clear summary",
    backstory="Skilled at explaining complex topics"
)

# Define tasks
research_task = Task(
    description="Research recent papers on RAG",
    expected_output="List of 3 papers with summaries",
    agent=researcher
)

write_task = Task(
    description="Write a 500-word summary",
    expected_output="A polished summary article",
    agent=writer
)

# Assemble crew
crew = Crew(
    agents=[researcher, writer],
    tasks=[research_task, write_task],
    process="sequential"
)

result = crew.kickoff()

The researcher runs first, finds the papers. Its output automatically flows to the writer. The writer uses those findings to produce the summary. Two agents, two tasks, one crew. That is multi-agent AI.

PART D Sequential vs Hierarchical
Voiceover + Animation ~2 min

Once you have your agents and tasks, you need to decide how they work together. CrewAI gives you two options.

The first is sequential. The researcher goes first, finishes, and hands everything to the writer. The writer finishes and hands everything to the reviewer. One after another, like a production line. You know exactly what happens and in what order.

The second is hierarchical. Instead of a fixed order, you add a manager agent. The manager reads the overall goal, decides which agent should work next, reviews the output, and can send it back if it is not good enough. It is closer to how a real project team operates.

For most tasks, sequential is the right starting point. You will know when you need hierarchical because sequential will start feeling too rigid.

Storyboard -- Sequential vs Hierarchical
PART E Delegation
Voiceover + Animation ~2 min

One of CrewAI's most powerful features is delegation. An agent can ask another agent for help.

The writer is drafting a section and realizes it needs a specific statistic. Instead of guessing, it delegates to the researcher: "Find me the 2024 market size for vector databases." The researcher runs, finds the number, and passes it back. The writer continues with real data.

This is not hardcoded. The agents decide when to delegate based on the task at hand. It is collaboration, not just sequencing.

Video 3: "Orchestrating Complex Workflows"

Voiceover + animation -- 6-8 minutes -- 3 segments
PART A Task Dependencies
Voiceover + Animation ~2 min

Real workflows have dependencies. The analyst cannot analyze until the researcher has gathered data. The writer cannot write until the analyst has produced insights. The editor cannot edit until the writer has a draft.

CrewAI handles this through task ordering and context passing. Each task's output becomes available to downstream tasks. You define the order, and the framework ensures that each agent gets the context it needs from the agents that ran before it.

Storyboard -- Crew Execution Timeline
PART B Memory and Context
Voiceover + Animation ~2 min

CrewAI also handles memory. When the researcher finishes, its output is automatically passed to the next agent as context. The analyst does not just see its own task description -- it sees everything the researcher found. And when the analyst finishes, the writer sees both the research and the analysis.

Think of it like a relay race. Each runner passes the baton to the next. The baton is the accumulated context -- and it gets richer at every handoff. You do not wire this up manually. You define the task order, and CrewAI handles the rest.

Storyboard -- Context Accumulates Down the Chain
PART C Real-World Patterns
Voiceover + Animation ~2 min

CrewAI is already running in production at companies like Uber, LinkedIn, and hundreds of others. The use cases follow the same pattern every time.

A content team where one agent researches, another writes, and a third edits. A lead scoring pipeline where one agent qualifies the prospect, another checks the CRM, and a human reviewer approves or rejects. An email responder where one agent reads the incoming message, classifies the intent, and routes it to a specialist agent that drafts the reply.

The pattern is always the same. Break the job into roles. Give each role one clear task. Let CrewAI handle the handoffs.

Video 5: "CrewAI Tools & Processes"

On camera + animation + code -- 5-7 minutes -- 3 segments
KEY MESSAGE Tools give agents superpowers. Processes decide who goes when. Together they turn a crew into a system.
PART A CrewAI Tools — Giving Agents Superpowers
On Camera + Animation ~2.5 min

In the last video you saw how to define agents, assign them roles, and wire them into a crew. But there is a piece we have not talked about yet -- and it changes everything.

Tools. A CrewAI tool is a Python function that an agent can call during its work. Search the web. Read a file. Query a database. Scrape a website. Send an email. Calculate a number. Without tools, an agent can only think. With tools, it can act.

CrewAI ships with a library of built-in tools -- SerperDevTool for web search, ScrapeWebsiteTool for pulling web content, FileReadTool for reading local files, and more. But the real power is custom tools. You write a Python function, decorate it with @tool, give it a good docstring, and the agent knows how to use it.

Here is the key insight: the quality of your tools determines the quality of your crew. An agent with bad tools is like a surgeon with dull instruments -- smart, but limited.

PART B Processes — Sequential vs Hierarchical
On Camera + Animation ~2 min

You already know the difference between sequential and hierarchical from the last video. Now let us talk about when you actually switch from one to the other.

Sequential breaks down when the task requires back-and-forth. If your editor finds a factual error, in sequential mode the crew is already done. There is no way to send it back to the researcher. In hierarchical mode, the manager catches that and says "researcher, verify this claim before we continue."

Hierarchical also shines when you do not know the right order in advance. Maybe some queries need research first, others need analysis first. The manager figures that out at runtime based on what the user asked.

The trade-off is cost and predictability. Hierarchical uses more LLM calls because the manager reasons at every step. And the output path is harder to debug because you did not design it. For production systems, start with sequential, measure quality, and only switch to hierarchical if you see the quality ceiling.

PART C Memory, Delegation & Guardrails
On Camera ~1.5 min

Three more features you should know about before the lab.

Memory. CrewAI can give agents short-term memory (within a task), long-term memory (across crew runs), and entity memory (facts about specific things). Turn it on with memory=True on the Crew. It makes a real difference when agents need to build on each other's work.

Delegation. In hierarchical mode, agents can delegate sub-tasks to other agents. The manager says "Agent B, handle this piece" and Agent B runs its own mini-workflow. You control this with allow_delegation=True on the agent.

Guardrails. You can set max_iter to limit how many reasoning steps an agent takes, and max_rpm to throttle API calls. Without these, a confused agent can loop forever and burn through your API budget. Always set limits.

Tools, processes, memory, delegation, guardrails. That is the full CrewAI toolkit. You have everything you need for the next lab.
