Video 2: "CrewAI: Building Agent Teams"

Voiceover + animation -- 10-12 minutes -- 5 segments
SEG A The Story Behind CrewAI
Voiceover + Animation ~2 min

CrewAI was created in late 2023 by Joao Moura, a Brazilian-born engineer who had spent years building AI systems at large companies. He noticed the same problem over and over -- developers were writing hundreds of lines of boilerplate just to get two agents to talk to each other.

His idea was simple. What if multi-agent systems worked the way real teams work? You define who is on the team, what each person is responsible for, and how they collaborate. The framework handles the rest.

He open-sourced it in early 2024, and it took off. Within months, CrewAI had tens of thousands of stars on GitHub and a growing community of developers building everything from research assistants to automated content pipelines. By 2025, CrewAI had raised funding and become one of the most popular multi-agent frameworks in the Python ecosystem.

What makes CrewAI different from building agents in LangGraph is the level of abstraction. LangGraph gives you nodes, edges, and state -- you build the graph yourself. CrewAI gives you agents, tasks, and crews -- you describe the team, and it builds the graph for you.

Storyboard -- CrewAI Timeline
SEG B The Three Building Blocks
Voiceover + Animation ~3 min

CrewAI has three building blocks. That is it. Agents, tasks, and crews.

An agent is a specialist. It has a role -- "Senior Researcher." It has a goal -- "Find the three most relevant papers on this topic." It has a backstory -- context that shapes how it behaves. And optionally, it has tools -- functions it can call.

A task is an assignment. It has a description -- what needs to be done. It has an expected output -- what the result should look like. And it is assigned to a specific agent.

A crew is the team. It takes a list of agents and a list of tasks, and it orchestrates the work. Sequential or hierarchical -- you choose the process.

SEG C The Code
Voiceover + Code ~3 min

Here is what it looks like in code.

from crewai import Agent, Task, Crew

# Define agents
researcher = Agent(
    role="Senior Researcher",
    goal="Find the 3 most relevant papers",
    backstory="Expert at academic research"
)

writer = Agent(
    role="Technical Writer",
    goal="Write a clear summary",
    backstory="Skilled at explaining complex topics"
)

# Define tasks
research_task = Task(
    description="Research recent papers on RAG",
    expected_output="List of 3 papers with summaries",
    agent=researcher
)

write_task = Task(
    description="Write a 500-word summary",
    expected_output="A polished summary article",
    agent=writer
)

# Assemble crew
crew = Crew(
    agents=[researcher, writer],
    tasks=[research_task, write_task],
    process="sequential"
)

result = crew.kickoff()

The researcher runs first, finds the papers. Its output automatically flows to the writer. The writer uses those findings to produce the summary. Two agents, two tasks, one crew. That is multi-agent AI.

SEG D Sequential vs Hierarchical
Voiceover + Animation ~2 min

CrewAI supports two orchestration patterns.

Sequential: tasks run in order. The researcher finishes, then the writer starts, then the reviewer. Output flows downstream. Simple, predictable, great for linear workflows.

Hierarchical: a manager agent delegates tasks, reviews outputs, and can reassign work. The manager decides the order. It can send work back for revision. More complex, more flexible, better for dynamic projects where the path is not fixed.

Start with sequential. Move to hierarchical when you need it.

Storyboard -- Sequential vs Hierarchical
SEG E Delegation
Voiceover + Animation ~2 min

One of CrewAI's most powerful features is delegation. An agent can ask another agent for help.

The writer is drafting a section and realizes it needs a specific statistic. Instead of guessing, it delegates to the researcher: "Find me the 2024 market size for vector databases." The researcher runs, finds the number, and passes it back. The writer continues with real data.

This is not hardcoded. The agents decide when to delegate based on the task at hand. It is collaboration, not just sequencing.

Video 3: "Orchestrating Complex Workflows"

Voiceover + animation -- 6-8 minutes -- 3 segments
SEG A Task Dependencies
Voiceover + Animation ~2 min

Real workflows have dependencies. The analyst cannot analyze until the researcher has gathered data. The writer cannot write until the analyst has produced insights. The editor cannot edit until the writer has a draft.

CrewAI handles this through task ordering and context passing. Each task's output becomes available to downstream tasks. You define the order, and the framework ensures that each agent gets the context it needs from the agents that ran before it.

Storyboard -- Crew Execution Timeline
SEG B Memory and Context
Voiceover + Animation ~2 min

CrewAI also handles memory. When the researcher finishes, its output is automatically passed to the next agent as context. The analyst does not just see its own task description -- it sees everything the researcher found. And when the analyst finishes, the writer sees both the research and the analysis.

Think of it like a relay race. Each runner passes the baton to the next. The baton is the accumulated context -- and it gets richer at every handoff. You do not wire this up manually. You define the task order, and CrewAI handles the rest.

Storyboard -- Context Accumulates Down the Chain
SEG C Real-World Patterns
Voiceover + Animation ~2 min

Where are multi-agent systems actually being used?

Cognition's Devin uses multiple specialized agents -- a planner, a coder, a debugger, and a tester -- working together to complete software engineering tasks. Valued in the billions.

Harvey uses multi-agent workflows for legal research -- one agent finds case law, another analyzes precedents, a third drafts arguments. Used by elite law firms.

The pattern is the same everywhere. Break a complex task into specialized roles. Let each role do what it does best. Orchestrate the handoffs. That is multi-agent AI.
