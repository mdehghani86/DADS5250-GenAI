PART 1: RECAP
------------------------

You built something this week that most engineers have not built yet.

You created a CrewAI crew where specialized agents collaborated on a task -- each with its own role, its own tools, its own goal. You built a LangGraph workflow where agents followed a directed graph, making decisions at branch points and passing state between nodes.

You learned that multi-agent is not about making one agent smarter. It is about making the system smarter by dividing the work.

The key insight: define clear roles, give each agent just enough scope, and design the handoffs carefully.

[ANIMATION: Quick montage -- CrewAI role cards, LangGraph node diagram, final output from both labs]

PART 2: WHAT IS NEXT
------------------------

Your agents can now work in teams. But they are still isolated from the outside world. They can only use the tools you explicitly give them.

In Module 9, we break that wall down. MCP -- the Model Context Protocol -- gives your agents a universal connector to any tool, any database, any service. And guardrails make sure they do it safely.

Your agents are about to get connected.
