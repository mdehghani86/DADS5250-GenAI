Video 2: Vision APIs -- Sending Images to AI

Voiceover + code walkthrough -- 6-8 minutes -- 3 segments
SEG A How to Send Images
Voiceover + Code ~3 min

Let me show you how to send an image to an AI model. It is simpler than you think.

With GPT-4 Vision, you send a message that contains both text and an image. The image can be a URL or a base64-encoded file. The model processes both together and responds.

With Gemini Vision, the approach is similar but the API structure differs slightly. You can send images, PDFs, even short video clips.

Code Display
Code Walkthrough Side-by-side: GPT-4V API call (left) vs Gemini Vision API call (right). Highlight the image parameter in both. Show the response with image analysis.
SEG B What Vision Models Can Do
Voiceover + Demos ~3 min

What can these models actually do with images? Object detection, text extraction from photos, diagram analysis, comparing two images, describing scenes, reading handwriting. The accuracy varies by task, but for many practical applications it is remarkably good.

Demo: Model Comparison

The key insight is this -- vision is not a separate capability. It is integrated into the same conversation. You can ask follow-up questions about an image. You can send multiple images and ask the model to compare them. It is a conversation, not a one-shot analysis.

Video 3: LLM Evaluation -- Metrics and Frameworks

Voiceover + diagrams -- 6-8 minutes -- 4 segments
SEG A The Hard Problem
Voiceover + Animation ~2 min

You have built a RAG system. You have built agents. But here is the question nobody asks until it is too late -- how do you know if they are working well?

LLM evaluation is one of the hardest problems in AI engineering. Traditional software has unit tests -- input X, expect output Y. But LLMs are non-deterministic. The same input can produce different outputs every time.

Animation
Animation Traditional testing: input arrow to function arrow to expected output (checkmark). LLM testing: same input, three different outputs fan out. Question mark -- which one is "correct"? ~10 sec
SEG B Automated Metrics
Voiceover + Diagrams ~2 min

So what do we measure? There are automated metrics like BLEU and ROUGE -- they compare generated text against reference text. Useful for translation and summarization. There is semantic similarity -- embedding the output and comparing it to expected answers. And there is human evaluation -- having people rate quality, accuracy, helpfulness.

Demo: Evaluation Dashboard
Dashboard Professional metrics visualization -- bar charts of model scores, confusion matrix, precision/recall curves. Like a testing dashboard for AI. ~10 sec
MEDIA TEAM: Clean dashboard layout. Top row: overall score cards. Middle: bar chart comparing BLEU, ROUGE, semantic similarity across models. Bottom: precision/recall curve. Use the course blue palette. No decorative elements -- this should feel like a real engineering tool.
SEG C RAG Evaluation with RAGAS
Voiceover + Code ~2 min

For RAG systems specifically, you measure retrieval quality (did we find the right documents?) and generation quality (did the answer correctly use those documents?). Frameworks like RAGAS give you metrics for both.

Code Display
Code Walkthrough RAGAS evaluation code: test dataset, retrieval metrics (context precision, recall), generation metrics (faithfulness, answer relevancy). Show the output scores.
SEG D Practical Approach -- Regression Testing for AI
Voiceover + Screen ~2 min

The practical approach? Build a test suite. A set of questions with known good answers. Run your system against it regularly. Track the scores over time. That is regression testing for AI.

Diagram
Flow Diagram Build test suite (questions + expected answers) arrow to Run system arrow to Compare outputs arrow to Track scores over time (line chart trending). Feedback loop.
