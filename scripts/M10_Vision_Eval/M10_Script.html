Video 2: Vision APIs -- Sending Images to AI

Voiceover + code walkthrough -- 8-10 minutes -- 5 segments
PART A How to Send Images
Voiceover + Code ~3 min

Sending an image to an AI model is simpler than you think.

With OpenAI's multimodal models, you send a message that contains both text and an image. The image can be a URL or a base64-encoded file. The model processes both together and responds.

With Gemini, the approach is similar but the API structure differs slightly. You can send images, PDFs, even short video clips.

Storyboard -- Sending an Image to a Multimodal API
PART B What Vision Models Can Do
Voiceover + Demos ~3 min

Vision models can do more than you might expect. Object detection, text extraction from photos, diagram analysis, comparing two images, describing scenes, reading handwriting. The accuracy varies by task, but for many practical applications it is remarkably good.

Storyboard -- What Vision Models Can Do
MEDIA TEAM: Use a complex image (engineering diagram or data-rich chart). Show both model outputs side by side. Highlight where they agree (green) and where they differ (amber). This sets up the evaluation discussion in V3.
PART C Conversational Vision
Voiceover + Screen ~2 min

The key insight is this. Vision is not a separate capability. It is integrated into the same conversation. You can ask follow-up questions about an image. You can send multiple images and ask the model to compare them. It is a conversation, not a one-shot analysis.

Demo

Show a multi-turn conversation: upload image, ask question, get answer, ask follow-up about a specific region, get detailed analysis. The AI remembers the image across turns.

PART D Vision API Costs
Voiceover + Animation ~2 min

Vision calls cost more than text calls. That is the trade-off. An image adds tokens to your request, and those tokens are not cheap.

With OpenAI, a single image can add anywhere from a few hundred to several thousand tokens depending on the resolution. High-detail mode costs more but gives better results for complex images like engineering diagrams. Low-detail mode is cheaper and works fine for simple photos.

With Gemini, the pricing is more generous. Google's free tier includes vision calls, which makes it a great option for prototyping and student projects. For production workloads, their paid tier is competitive.

The practical lesson is this. Do not send high-resolution images when a thumbnail would do. Resize before you send. Use low-detail mode for classification tasks and high-detail mode only when you need precision. A little optimization goes a long way on your API bill.

Storyboard -- Vision API Cost Comparison
PART E When Vision Makes Sense
Voiceover + Animation ~90 sec

Not every problem needs vision. If the data is already in text form, sending a screenshot of a spreadsheet to a vision model is a waste of money. Just send the CSV.

Vision makes sense when the information is inherently visual. A photo of a damaged product. An engineering diagram. A handwritten form. A chart someone pasted into a slide deck without sharing the data. Those are the cases where text alone cannot capture what you need, and vision becomes the right tool.

Video 4: LLM Evaluation -- Metrics and Frameworks

Voiceover + diagrams -- 6-8 minutes -- 4 segments
PART A The Hard Problem
Voiceover + Animation ~2 min

You have built a RAG system. You have built agents. But here is the question nobody asks until it is too late. How do you know if they are working well?

LLM evaluation is one of the hardest problems in AI engineering. Traditional software has unit tests. Input X, expect output Y. But LLMs are non-deterministic. The same input can produce different outputs every time.

Storyboard -- Why Testing LLMs Is Hard
PART B Automated Metrics
Voiceover + Diagrams ~2 min

Evaluation starts with knowing what to measure. There are automated metrics like BLEU and ROUGE. They compare generated text against reference text. Useful for translation and summarization. There is semantic similarity, embedding the output and comparing it to expected answers. And there is human evaluation, having people rate quality, accuracy, helpfulness.

Storyboard -- Eval Metrics Dashboard
MEDIA TEAM: Clean dashboard layout. Top row: overall score cards. Middle: bar chart comparing BLEU, ROUGE, semantic similarity across models. Bottom: precision/recall curve. Use the course blue palette. No decorative elements -- this should feel like a real engineering tool.
PART C RAG Evaluation with RAGAS
Voiceover + Code ~2 min

For RAG systems specifically, you measure retrieval quality (did we find the right documents?) and generation quality (did the answer correctly use those documents?). Frameworks like RAGAS give you metrics for both.

Storyboard -- RAGAS Metrics
PART D Practical Approach -- Regression Testing for AI
Voiceover + Screen ~2 min

The practical approach? Build a test suite. A set of questions with known good answers. Run your system against it regularly. Track the scores over time. That is regression testing for AI.

Storyboard -- Regression Testing for AI
