SEG 01 Look Around -- It's Already Here

Open up Google Chrome today, and there is a little button that summarizes any news article for you in about two seconds.

Open Google Maps, and it no longer just gives you directions -- it answers questions like "find me a quiet coffee shop near my hotel that opens before seven."

And if you booked a flight recently, you may have noticed something new in your confirmation email: an AI-generated itinerary, ready to use the moment you land.

None of these features existed two years ago. Today, they are just... there. Showing up quietly inside the tools you already use.

Here is what is happening, and it is happening fast. Every serious software company on the planet is asking the same question right now -- and it is not "should we add AI?" That question is over.

The question now is "where do we add it, and how soon?" Chrome added it to reading. Maps added it to search. Airlines added it to travel planning. Photoshop added it to editing. Notion added it to writing. Slack added it to summarizing conversations.

The common thread is this: AI is not becoming a product on its own. It is becoming a layer. A layer that sits inside every tool you already use and quietly makes it smarter.

And that is where this module begins. If the biggest companies in the world are racing to embed AI into their tools, there is no reason you cannot start embedding it into yours.

The spreadsheet you already open every morning. The financial analysis you already do by hand. The workflow you already have. These are not toy problems -- these are the things you work on. And they are exactly where AI belongs next.

This module is not about building a demo. It is about sparking an idea -- the idea that the moment you see how simple it is to drop AI into something that already exists, you stop asking if you can build it. You start asking what you will build next.

Video 2: "Excel GPT -- AI Meets Spreadsheets"

Voiceover + animation + code -- 6-8 minutes -- 4 segments
KEY MESSAGE Describe what you want in English. Let the AI write the formulas.
SEG A Why Excel + GenAI
On Camera ~30 sec

Let me start with something personal. I firmly believe Excel is one of the most powerful tools ever built for business. Data analysis. Visualization. Reporting. Modeling. You name it -- Excel can do it. That is why nearly every organization on the planet runs on spreadsheets. It is the silent workhorse of how work actually gets done.

So when I started building this course, adding generative AI to Excel was a no-brainer. If we can make Excel smarter, faster, and easier to work with, we touch the daily workflow of millions of professionals. That is where this module starts.

Storyboard -- Why Excel Matters
MEDIA TEAM: Reveal the green Excel card first as Prof says "Excel is one of the most powerful tools." Then the four use-case tiles slide in as he lists them ("analysis, visualization, reporting, modeling"). Then the "+" symbol appears, then the blue "Generative AI" card fades in with "= no-brainer" as the punchline. Warm, confident, not rushed.
SEG B The Problem -- Manual Excel Work
Voiceover + Animation ~90 sec

Here is the problem most of us know too well. You have a spreadsheet with sales data -- thousands of rows. You need to calculate monthly growth rates, create a pivot summary, and highlight outliers. Normally, you would spend an hour writing formulas. Remembering the exact syntax. Debugging typos. Copying formulas down thousands of rows. By the time you are done, you have forgotten what question you were trying to answer in the first place.

With an API call, this changes. You describe what you want to the model in plain English: "Analyze this sales data. Calculate month-over-month growth rates. Flag any month where growth dropped below negative ten percent. Generate the Excel formulas I need."

The model responds with working formulas. You paste them into Excel. Done.

Storyboard -- Manual vs AI
MEDIA TEAM: The emotional arc is frustration (left) to relief (right). The manual side should feel painfully slow -- real typing sounds, clock ticking. The AI side should feel effortless -- a whoosh sound, cells filling like dominoes. The time comparison counter is the punchline.
SEG C Extend Excel with VBA + AI
Voiceover + Code Display ~2 min

Now here is where it gets really interesting. If you know Excel well, you know that you can go beyond formulas -- you can extend its capabilities with VBA. Visual Basic for Applications is Excel's built-in scripting language. With VBA, you can build custom functions, automate tasks, format data, generate charts, even connect to external systems. VBA is what turns Excel from a spreadsheet into a mini application platform.

The catch? Most people do not know VBA. It is an older language, the syntax is clunky, and writing it from scratch is slow and painful.

This is where the same API call pattern unlocks something big. You can ask the AI to write the VBA for you. "Write a VBA macro that automatically formats the top 10 products by revenue in bold and adds a chart." The model writes the macro. You paste it into Excel's VBA editor. Run it. Your spreadsheet transforms.

In Lab 1, you will build exactly this from scratch. The starter code is on GitHub. Your assignment is to extend it with your own functions.

Storyboard -- English → VBA → Excel
MEDIA TEAM: The morph from code to spreadsheet is the key moment. The code should not just disappear -- it should visually flow into the Excel window, like pouring water into a glass. The chart growing from the data is the satisfying payoff. Consider a subtle camera zoom as the chart completes.

TA Excel GPT Lab Walkthrough
TA GUIDELINES 8-12 min

GUIDELINES FOR TA

Open the Excel GPT Colab notebook and record a live screen walkthrough. Run every cell from top to bottom. Talk naturally -- no script. Friendly, patient, at a pace a first-time student can follow.

1. Open the Notebook
  • Open M02_Lab1_ExcelGPT.ipynb in Google Colab
  • Show the table of contents and explain the flow: setup → prompt → generate → paste into Excel
  • Confirm the API key is already loaded from Colab Secrets
2. Walk Through the Code Cells
  • Cell by cell, explain what each line does in plain English
  • Pause on the prompt template -- highlight how the "role" and "user" messages are structured
  • Run each cell live and show the output
  • If a cell fails, fix it on camera -- students learn from seeing errors resolved
3. Demo the Tool End-to-End
  • Take a sample request: "Write a VLOOKUP that pulls the price from Sheet2 where the SKU matches column A"
  • Run it and show the generated formula
  • Paste it into a real Excel file and show it working
  • Do at least 2 more examples of increasing complexity (pivot formula, VBA macro)
4. Common Gotchas
  • Show what happens when the prompt is too vague -- and how to fix it
  • Show a response with an incorrect formula and how to spot it
  • Remind students: always test AI-generated code before trusting it
5. Assignment Handoff
  • Point to the exercise at the bottom of the notebook
  • Tell students what to submit and where
  • Sign off warmly: "Good luck -- and have fun with it."

TA screen recording at 1920x1080, 30fps. Colab notebook at comfortable zoom (students should be able to read every line). TA voiceover recorded with a decent mic -- clean, warm, conversational.

MEDIA TEAM: This is a walkthrough, not a scripted lecture. Record the screen, capture the audio, edit only for length (cut long pauses and mistakes). No B-roll, no overlays. Keep it raw and authentic -- students trust walkthrough videos more than polished ones.
