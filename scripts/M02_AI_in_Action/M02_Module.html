Here is something that will change how you think about Excel.

You have a spreadsheet with sales data -- thousands of rows. You need to calculate monthly growth rates, create a pivot summary, and highlight outliers. Normally, you would spend an hour writing formulas, maybe some VBA if it gets complex.

Instead, you describe what you want to the API: "Analyze this sales data. Calculate month-over-month growth rates. Flag any month where growth dropped below negative ten percent. Generate the Excel formulas I need."

The model responds with working formulas. You paste them into Excel. Done.

But here is the real power -- you can also ask it to write VBA macros. "Write a VBA macro that automatically formats the top 10 products by revenue in bold and adds a chart." The model writes the macro. You paste it into Excel's VBA editor. Run it. Your spreadsheet transforms.

In Lab 1, you will build this from scratch. The starter code is on GitHub. Your assignment is to extend it with your own functions.

Now let us build something completely different.

Cryptocurrency markets generate massive amounts of data -- prices, volumes, trends, all changing by the second. Analyzing this data manually is tedious and slow.

But with the API, you can feed real Bitcoin price data to the model and ask it to identify patterns, calculate technical indicators, and generate an investment analysis report -- all in a few lines of Python.

You will load real market data using a free API. You will send it to GPT along with instructions for analysis. And the model will return a structured report with trend analysis, support and resistance levels, and a summary recommendation.

Is it financial advice? Absolutely not. But it is a working demonstration of how AI can process real data and generate structured analysis -- the same pattern used in every industry from finance to manufacturing to healthcare.

Vibe Coding: The New Way to Build Software

Vibe coding means describing what you want in plain English and letting AI write the code. You guide the direction. AI handles the syntax.

The Shift

For decades, building software meant memorizing syntax, debugging semicolons, and reading documentation. Vibe coding flips this model. You describe the outcome you want -- "create a dashboard that shows monthly sales trends" -- and AI generates the code. Your job is to evaluate, iterate, and improve.

The Evidence

Productivity
GitHub Copilot users complete tasks 55% faster (GitHub Research, 2022)
Adoption
Companies are actively hiring for "AI-assisted development" roles
Industry Shift
Google: 25%+ of new code is AI-generated (Sundar Pichai, Q3 2024)
The Term
Coined by Andrej Karpathy (Feb 2025) -- "just see stuff, say stuff, run stuff"

From Vibe Coding to Production

You just built two working tools by describing what you wanted to an AI. That is vibe coding. But there is a gap between a prototype and a product.

What You Just Did

You sent natural language to an API. The AI generated code, formulas, analysis. You ran it, and it worked. This is the power of vibe coding -- rapid prototyping, instant results, zero boilerplate.

The Gap

But you also noticed something. Sometimes the output was perfect. Other times, the format shifted. A formula used the wrong column reference. The analysis missed a key data point. The VBA code had a syntax error.

This is the gap between a vibe-coded prototype and a production system: reliability, consistency, error handling.

Vibe Coding (Now)

  • Fast prototyping
  • Natural language input
  • Output varies each run
  • Manual quality checks

Production (M03+)

  • Consistent output format
  • Engineered prompts
  • Same quality every run
  • Automated validation

"Next module -- Prompt Engineering -- teaches you to close that gap. You will learn to write prompts that produce the same quality output every time, not just when you are lucky."
