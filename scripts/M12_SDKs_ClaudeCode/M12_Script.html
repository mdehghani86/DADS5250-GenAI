SEG 02 The Platform Shift
On Camera ~60 sec

In 2024, building an AI agent meant choosing a framework, writing glue code, debugging tool integrations, and hoping everything worked together. LangChain, CrewAI, AutoGen -- each had strengths, but all required significant scaffolding.

In 2026, the major AI companies released their own SDKs. Anthropic shipped the Claude Agent SDK with built-in tools -- Read, Write, Bash, WebSearch. Google released the Agent Development Kit with multi-agent support and deployment flexibility. These are not frameworks. They are platforms.

Video 2: Claude Agent SDK

Voiceover + code walkthrough -- 6-8 minutes -- 3 segments
SEG A Built-In Tools
Voiceover + Code ~3 min

The Claude Agent SDK is Anthropic's official toolkit for building AI agents. Let me walk you through what makes it different.

First, built-in tools. The SDK comes with tools like Read (read files), Write (create files), Bash (run commands), and WebSearch. You do not need to implement these yourself. They just work.

Demo: SDK Comparison
Three Columns Claude SDK, OpenAI SDK, Google ADK. Each with code snippets showing how to build the same agent. Highlight differences and similarities. Hold 15 sec
MEDIA TEAM: Three-column layout. Same task in each SDK. Highlight lines of code needed (Claude = fewest due to built-in tools). Use the course blue palette for Claude column, neutral for others.
SEG B Skills and Subagents
Voiceover + Diagrams ~3 min

Second, Skills versus Tools. Tools are atomic actions. Skills are higher-level capabilities -- reusable instruction sets that the agent can invoke. Think of tools as functions and skills as libraries.

Third, subagents. You can create specialized agents that work under a main orchestrator. Each subagent has its own tools and instructions. The orchestrator delegates tasks and collects results.

Architecture Diagram
Diagram Orchestrator agent at top. Three subagents below (Research, Analysis, Writing). Each with their own tool icons. Arrows show delegation and result collection.
SEG C Mental Model
Voiceover + Screen ~2 min

The mental model is simple -- you define what the agent should do (instructions), what it can do (tools), and how it should work with others (subagents). The SDK handles the rest.

Demo: Claude Code
Screen Recording Terminal showing Claude Code building a feature in real-time. Commands flow, files get created, tests pass. Show the power of agent-driven development. ~20 sec

Video 3: Google ADK

Voiceover + code walkthrough -- 5-6 minutes -- 3 segments
SEG A Code-First Approach
Voiceover + Code ~2 min

Google's Agent Development Kit takes a code-first approach. If you are a developer who thinks in functions and classes, ADK will feel natural.

The core concept is the Agent class. You define an agent with a name, model, instructions, and tools. Tools are Python functions decorated with metadata. The agent calls them as needed.

Code Display
Code Walkthrough Google ADK Agent class definition. Show the Agent constructor with name, model, instructions, tools. Highlight the decorator pattern for tools.
SEG B Multi-Agent Orchestration
Voiceover + Diagrams ~2 min

What makes ADK interesting is multi-agent orchestration. You can define multiple agents that collaborate on complex tasks. A research agent gathers information. An analysis agent processes it. A writing agent produces the final output.

Diagram
Flow Diagram Three agents in sequence: Research arrow to Analysis arrow to Writing. Data flows between them. Each agent box shows its tools and model.
SEG C Deployment Flexibility
Voiceover + Screen ~2 min

Deployment is flexible -- run locally, deploy to Cloud Run, or use Vertex AI. Same code, different environments.

Deployment Options
