REVIEW -- First Draft

The Claude Agent SDK is Anthropic's official toolkit for building AI agents. Let me walk you through what makes it different.

First, built-in tools. The SDK comes with tools like Read (read files), Write (create files), Bash (run commands), and WebSearch. You do not need to implement these yourself. They just work.

Second, Skills versus Tools. Tools are atomic actions. Skills are higher-level capabilities -- reusable instruction sets that the agent can invoke. Think of tools as functions and skills as libraries.

Third, subagents. You can create specialized agents that work under a main orchestrator. Each subagent has its own tools and instructions. The orchestrator delegates tasks and collects results.

The mental model is simple -- you define what the agent should do (instructions), what it can do (tools), and how it should work with others (subagents). The SDK handles the rest.

REVIEW -- First Draft

Google's Agent Development Kit takes a code-first approach. If you are a developer who thinks in functions and classes, ADK will feel natural.

The core concept is the Agent class. You define an agent with a name, model, instructions, and tools. Tools are Python functions decorated with metadata. The agent calls them as needed.

What makes ADK interesting is multi-agent orchestration. You can define multiple agents that collaborate on complex tasks. A research agent gathers information. An analysis agent processes it. A writing agent produces the final output.

Deployment is flexible -- run locally, deploy to Cloud Run, or use Vertex AI. Same code, different environments.

The SDK Revolution

In 2024, building an AI agent required a framework, glue code, and patience. In 2026, the AI companies shipped their own SDKs -- and everything changed.

The Old Way (2024)

  • Choose a framework (LangChain, CrewAI, AutoGen)
  • Write custom tool implementations
  • Build integration layers between components
  • Debug framework-specific issues
  • Hope version updates do not break everything

The New Way (2026)

  • Import the SDK
  • Define your agent (model, instructions, tools)
  • Run it

The shift is not just about convenience. First-party SDKs have deeper model integration, better error handling, and native support for features like tool use and multi-turn conversations. They are maintained by the same teams that build the models.

Timeline

YearDevelopment
2023LangChain dominates. Agents are experimental.
2024CrewAI, AutoGen emerge. Multi-agent is the trend.
2025OpenAI Agents SDK, Anthropic Claude Agent SDK launch.
2026Google ADK ships. First-party SDKs become the standard.

LangChain, CrewAI, and LangGraph are not dead -- they serve different purposes. But for most new projects, starting with a first-party SDK is the pragmatic choice.

Claude Code as an Engineering Tool

Claude Code is Anthropic's agentic coding tool -- a terminal-based AI that reads your codebase, writes code, runs commands, and iterates. It is not an autocomplete. It is a collaborator.

How Prof. Dehghani Uses It Daily

Every project in this course was built with Claude Code. The module players you are reading right now. The lab notebooks. The course outline. Not by copying and pasting from ChatGPT -- by working with an AI that understands the full codebase, maintains context across sessions, and can execute code to verify its own work.

The Shift: IDE to AI Pair Programming

Traditional development: you write code in an IDE, test it, debug it, repeat. AI pair programming: you describe what you want, the AI writes it, runs it, and iterates until it works. Your role shifts from writing code to reviewing code and making design decisions.

What Makes Claude Code Different

  • Full codebase awareness: It reads your entire project, not just the current file
  • Tool use: It runs commands, reads files, searches the web -- not just generates text
  • Persistent context: Memory files and project context carry across sessions
  • Slash commands: Custom skills you define for repetitive tasks

This is not about replacing developers. It is about amplifying them. The engineers who learn to work effectively with AI coding tools will build faster, with fewer bugs, and at a scale that was previously impossible for small teams.

Comparing SDKs -- Decision Matrix

With five major options for building AI agents, choosing the right one depends on your use case, team, and deployment requirements.

SDK / FrameworkBest ForBuilt-in ToolsMulti-Agent
Claude Agent SDKFull-stack agents, file ops, codingRead, Write, Bash, WebSubagents
OpenAI Agents SDKGPT-powered agents, tool useCode Interpreter, SearchHandoffs
Google ADKMulti-agent pipelines, GCP deployCustom Python toolsNative
LangGraphComplex stateful workflowsVia LangChain toolsGraph-based
CrewAIRole-based team simulationCustom toolsCrew model

When to Use What

  • New project, Claude model: Claude Agent SDK
  • New project, GPT model: OpenAI Agents SDK
  • Multi-model or GCP deployment: Google ADK
  • Complex state machines: LangGraph
  • Role-based simulation: CrewAI

The honest answer for most projects: start with the first-party SDK for your chosen model. Only reach for a framework when the SDK cannot handle your specific orchestration needs.
