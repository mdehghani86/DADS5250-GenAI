M07 Production Script DADS 5250 -- Generative AI in Practice

Video 1: "The Spark" -- Build Something Real

Prof on camera -- 3-4 minutes -- 3 segments
KEY MESSAGE Build something real. Break something. Ship it.
SEG 01 The Shift
On Camera ~60 sec Medium close-up

For six weeks, I have been teaching you. This week, you teach me.

No new lectures. No new frameworks. No new concepts. Just you, your team, forty-eight hours, and everything you have learned so far. LLM APIs. Prompt engineering. Structured output. LangChain. RAG. Agents. All of it. Combined into one project. Something real.

This is the hackathon.

Shot Setup

Medium close-up, slightly more energized than usual. Direct to camera. Build momentum through the list of skills. Pause before "This is the hackathon."

MEDIA TEAM: Clean shot, no overlays. Energy should feel different from typical lecture -- more like a coach before a game.
SEG 02 Startup Inspiration
On Camera ~60 sec

Let me give you some perspective on what is possible.

Jasper AI -- built on GPT API calls to generate marketing copy. Grew to tens of millions in annual revenue within its first couple of years. Cursor -- an AI code editor built on LLM APIs plus RAG over codebases. Raised hundreds of millions from top venture firms. Perplexity -- RAG applied to the entire internet. Now a multi-billion-dollar company.

Every one of these companies started with the same tools you have right now. An API key and an idea. The difference is they shipped it.

Your hackathon project will not be the next Perplexity. But the process of building, breaking, and shipping? That is exactly the same.

Storyboard -- They All Started With an API Key
SEG 03 Key Message
On Camera ~20 sec

Build something real. Break something. Ship it. The clock starts now.

Countdown Card
MEDIA TEAM: Timer overlay appears as Prof says "the clock starts now." Hold for 4 seconds, then transition to title card.

Video 2: "Hackathon Kickoff"

Prof on camera -- 5-6 minutes -- 4 segments
SEG 01 Logistics
On Camera ~90 sec

Here are the rules.

Teams of two to three students. Forty-eight hours from kickoff to submission. You submit two things: a GitHub repository with working code and a three-minute video demo.

Your project must use at least two course techniques -- that means some combination of API calls, prompt engineering, structured output, LangChain, RAG, or agents. It must be Python-based and it must actually work. The judges will run your code.

Office hours are running continuously during the sprint. Slack channel is live for quick questions and team coordination.

Storyboard -- The Rules at a Glance
SEG 02 Project Ideas
On Camera ~60 sec

Not sure what to build? Here are categories that have produced strong projects.

AI Assistant -- a domain-specific chatbot with RAG. Legal research, medical FAQ, course advisor. Content Pipeline -- a chain that takes raw input and produces polished output. Blog posts, reports, social media threads. Data Analyzer -- upload a CSV, ask questions in plain English, get charts and insights. Code Tool -- an agent that reads a codebase and answers questions or generates tests. Workflow Automator -- connect multiple APIs to solve a real business process.

Pick something you care about. The best projects come from real problems, not hypothetical ones.

Storyboard -- 5 Project Categories
SEG 03 Judging Criteria
On Camera ~60 sec

You are judged on four things, equally weighted.

Technical depth -- meaningful use of two or more course techniques. Not just an API call. Show me architecture. Creativity -- original problem choice. Surprise us. Build something we have not seen before. Polish -- it works, no crashes, clean code, good error handling. And presentation -- a clear three-minute video where you explain what you built and why it matters.

Twenty-five percent each. Notice that presentation counts as much as technical depth. If you build something brilliant and cannot explain it, that is a problem.

Storyboard -- Judging Rubric (4 × 25%)
SEG 04 Final Words
On Camera ~30 sec

One more thing. The goal is not perfection. The goal is momentum. Build the simplest version that works. Get it running. Then improve it. Do not spend thirty hours planning and eighteen hours panicking.

Build early. Test often. Ship it. Good luck.

Transition
MEDIA TEAM: End on a high note. Consider a quick montage cut of students working, coding, presenting from stock footage. Then the 48:00:00 countdown appears again as the final frame.
