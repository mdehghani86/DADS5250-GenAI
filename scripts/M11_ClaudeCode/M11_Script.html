Video 2: "Introduction to Claude Code"

Prof on camera -- ~5 minutes -- 3 parts
PART A History and Motivation
On Camera ~2 min

Anthropic released Claude Code as a command-line tool and an IDE extension. The idea behind it was simple but important: AI should understand your entire project, not just the file you have open.

Most coding assistants in 2024 worked file by file. You would paste a snippet, get a suggestion, paste it back. The AI never saw your directory structure, your test suite, or your git history. It was helpful, but it was limited.

Claude Code took a different approach. It runs inside your terminal or your editor. It scans your codebase. It builds a model of how your files connect. And then it acts on that understanding. It does not just suggest code. It writes files, runs commands, and iterates until the job is done.

The developer community adopted it fast. Engineers started using it for debugging, feature development, refactoring, and code review. It became one of the most talked-about developer tools of 2025.

PART B What Claude Code Can Do
On Camera ~1.5 min

Claude Code reads files, writes code, runs terminal commands, creates and edits files, searches the web, and manages git. It installs packages, executes tests, creates commits, and handles branches.

The key difference from a chatbot: it has access to your codebase. It sees everything. It reasons about your project as a whole. Changing a function signature in one file means updating every caller in three other files, and Claude Code knows that. It knows your test suite expects a specific return type. It traces dependencies across your entire project.

The difference between a coding chatbot and Claude Code is the difference between a spell checker and a co-author.

Storyboard -- Claude Code Capabilities
MEDIA TEAM: Animate capabilities appearing one by one around the central Claude Code box. Each lights up as Prof names it. End with the spell checker vs co-author comparison.
PART C The Trust Model
On Camera ~1.5 min

Claude Code proposes changes. You approve them. It asks before running commands, before editing files, before pushing code. You stay in control.

Every action goes through a permission step. Claude Code shows you exactly what it plans to do: the file it wants to edit, the command it wants to run, the commit message it wants to use. You approve or reject each one. Nothing happens without your explicit consent.

You can configure trust levels. Allow all file reads but require approval for writes. Allow test execution but block pushes to remote repositories. The default is conservative: ask before everything. You relax the controls as you build confidence.

The best way to use it is to treat it like a junior engineer who is very fast but needs supervision. It proposes. You decide.

TA Setup & Installation Walkthrough
TA GUIDELINES 8-10 min

GUIDELINES FOR TA

Walk students through installing Claude Code from scratch. Screen record at 1920x1080. Talk naturally, no script. Friendly, patient, at a pace a first-time student can follow.

1. Install the CLI
  • Show how to install Claude Code via npm install -g @anthropic-ai/claude-code
  • Verify the installation with claude --version
  • Mention Node.js prerequisite if students do not have it
2. Connect the API Key
  • Show how to set the Anthropic API key
  • Explain the difference between direct API key and OAuth login
  • Verify the connection works with a simple prompt
3. Open a Project
  • Navigate to a sample project directory in the terminal
  • Type claude and show the initialization process
  • Point out how Claude Code scans the project structure
4. First Interaction
  • Ask Claude Code a simple question about the project ("what does this project do?")
  • Show the permission prompt when it tries to read files
  • Demonstrate how to approve and reject actions
5. VS Code Extension Setup
  • Show how to install the Claude Code extension in VS Code
  • Demonstrate opening the Claude Code panel
  • Run the same task from the IDE and compare the experience
6. Checklist
  • CLI installed and verified
  • API key configured
  • First session completed successfully
  • VS Code extension installed (optional)
  • Sign off: "You are set up. The next lab puts it to real use."
Recording Setup

TA screen recording at 1920x1080, 30fps. Terminal at comfortable font size (students should be able to read every line). TA voiceover recorded with a decent mic. Clean, warm, conversational.

GUIDELINES FOR TA

Open a sample project with messy code and a pull request to review. Record a live screen walkthrough demonstrating code review, refactoring, and advanced patterns. Talk naturally, no script.

1. Code Review Workflow
  • Show a pull request or a set of changed files
  • Ask Claude Code: "Review these changes for bugs, security issues, and style problems."
  • Walk through the structured review output
  • Show how to iterate: "Fix the security issue you found in the auth middleware."
2. Refactoring a Messy File
  • Open a file with a long function (100+ lines)

  • Show how it identifies logical boundaries and extracts helpers
  • Run tests to verify nothing broke
