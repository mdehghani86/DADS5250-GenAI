Video 2: Gradio -- Instant AI Demos

Prof on camera + screen recording -- 6-8 minutes -- 4 segments
SEG A The Gradio Idea
On Camera ~90 sec

The idea behind Gradio is simple: take any Python function and turn it into a web interface automatically.

Here is how it works. You write a function. You wrap it in gr.Interface. You call launch. That is it. Gradio looks at your function signature, creates input components for the parameters, creates an output component for the return value, and gives you a local URL.

Storyboard -- Gradio in 7 Lines
MEDIA TEAM: Record VS Code or Colab on left, browser preview on right. Type the code live. When gr.Interface.launch() runs, the app should appear instantly. Highlight the share URL when it appears.
SEG B The Share Link Magic
Voiceover + Screen ~60 sec

But here is the real magic: add share=True and Gradio creates a public URL. Anyone in the world can use your app. No deployment. No server. Just a link.

Gradio has components for everything: text, images, audio, video, files, code, chatbots. You can build a text analyzer, a chatbot, an image classifier -- all with the same pattern.

Shot Setup

Show the terminal output with the share URL highlighted. Open the URL in a different browser or incognito window to prove it works anywhere.

SEG C Gradio Blocks
Voiceover + Code ~90 sec

For more complex apps, Gradio Blocks gives you full layout control. Rows, columns, tabs, accordions. You design the interface, wire up the events, and Gradio handles the rest.

Storyboard -- Gradio Blocks · Layout Control

Video 3: Streamlit -- AI Dashboards

Prof on camera + screen recording -- 6-8 minutes -- 4 segments
SEG A Streamlit Philosophy
On Camera ~90 sec

Streamlit was created in 2018 by Adrien Treuille, Amanda Kelly, and Thiago Teixeira. It takes a different approach than Gradio. Instead of wrapping functions, you write a script. The script runs top to bottom. Every Streamlit call adds something to the page.

st.title adds a title. st.write adds text. st.line_chart adds a chart. st.text_input adds an input box. It reads like English.

Storyboard -- Streamlit · Code Reads Like English
MEDIA TEAM: Split screen. Left: code being typed line by line. Right: dashboard building in real time. Each st.* call triggers a new UI element to appear on the right. Smooth slide-in animations.
SEG B State and Caching
Voiceover + Code ~90 sec

What makes Streamlit powerful for AI is its session state and caching. Session state remembers user inputs across reruns. Caching prevents expensive API calls from running every time the page refreshes. Together, they make it possible to build responsive AI dashboards.

Storyboard -- Caching: 5 Calls → 1 Call
SEG C Deployment
Voiceover + Screen ~60 sec

Streamlit also has a sidebar, columns, tabs, and metrics -- all the building blocks of a professional dashboard. And Streamlit Community Cloud lets you deploy for free. Push to GitHub, connect Streamlit Cloud, and your app is live.

Shot Setup

Show Streamlit Community Cloud deployment flow: GitHub push, connect repo, app goes live. Highlight the public URL.

SEG D Gradio vs Streamlit
On Camera ~30 sec

If Gradio is for demos, Streamlit is for products. Both are essential tools in your toolkit.

Storyboard -- Gradio for Demos · Streamlit for Products

TA Gradio Apps Lab Walkthrough
TA GUIDELINES 8-12 min

GUIDELINES FOR TA

Open M09_Lab1_Gradio_Apps.ipynb in Google Colab and record a live screen walkthrough. Run every cell from top to bottom. Talk naturally — no script. Friendly, patient, at a pace a first-time student can follow.

1. Open the Notebook
  • Open M09_Lab1_Gradio_Apps.ipynb in Google Colab
  • Show the table of contents and explain the flow of the lab
  • Confirm any required API keys are loaded from Colab Secrets
2. Walk Through the Code Cells
  • Cell by cell, explain what each line does in plain English
  • Run each cell live and show the output
  • If a cell fails, fix it on camera — students learn from seeing errors resolved
3. Highlight the Key Concept
  • Pause on the most important idea in this lab and explain why it matters
  • Connect it back to what the instructor covered in the scripted video right before this one
  • Show a second mini-example if there is time
4. Common Gotchas
  • Point out 2–3 places where students typically get stuck in this lab
  • Show the fix for each one
  • Remind students to read error messages carefully
5. Assignment Handoff
  • Point to the exercise at the bottom of the notebook
  • Tell students what to submit and where
  • Sign off warmly: "Good luck — and have fun with it."
Recording Setup

TA screen recording at 1920x1080, 30fps. Colab notebook at comfortable zoom (students should be able to read every line). TA voiceover recorded with a decent mic — clean, warm, conversational.

MEDIA TEAM: Walkthrough, not a scripted lecture. Record the screen, capture the audio, edit only for length. No B-roll, no overlays. Keep it raw and authentic — students trust walkthrough videos more than polished ones.
