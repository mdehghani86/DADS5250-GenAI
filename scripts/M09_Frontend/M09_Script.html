Video 2: Gradio -- Instant AI Demos

Prof on camera + screen recording -- 6-8 minutes -- 4 segments
PART A Gradio Origins
On Camera ~90 sec

Gradio started in 2019 when Abubakar Abid, a Stanford PhD student, got tired of showing his machine learning models through Jupyter notebooks. He wanted a way to turn any Python function into something anyone could use. Hugging Face acquired Gradio in 2021, and today it is the most popular way to demo AI models in Python.

The core idea is almost too simple. You write a Python function. You tell Gradio what the input is and what the output is. You call launch. Gradio builds the interface, starts a local server, and gives you a URL. Three lines of code and you have a working web application that anyone on your network can open in a browser.

Storyboard -- Gradio in 7 Lines
MEDIA TEAM: Record VS Code or Colab on left, browser preview on right. Type the code live. When gr.Interface.launch() runs, the app should appear instantly. Highlight the share URL when it appears.
PART B The Share Link Magic
Voiceover + Screen ~60 sec

One of the things that made Gradio so popular is sharing. With a single setting, Gradio generates a public link to your app. You can text that link to your boss, email it to a client, or post it in Slack. They open it in a browser and use your AI instantly. No server setup. No deployment pipeline. The link just works.

Gradio also supports far more than text. Images, audio, video, file uploads, even full chatbot interfaces. The lab will walk you through several of these, but the pattern is always the same. Write the function. Tell Gradio the input and output types. Launch.

Shot Setup

Show the terminal output with the share URL highlighted. Open the URL in a different browser or incognito window to prove it works anywhere.

PART C Gradio Blocks
Voiceover + Code ~90 sec

When you need more control over the layout, Gradio has a second mode called Blocks. Instead of one function and one interface, you design the page yourself. Rows, columns, tabs, accordions. You place each component exactly where you want it, wire up the button clicks, and Gradio handles the rendering. Think of Interface as the quick demo mode and Blocks as the full design mode.

Video 4: Streamlit -- AI Dashboards

Prof on camera + screen recording -- 6-8 minutes -- 4 segments
PART A Streamlit Philosophy
On Camera ~90 sec

Streamlit was created in 2018 by Adrien Treuille and his team. They came from Google Brain and wanted to build a tool where data scientists could create apps without learning web development. The result is a framework where you write a Python script and every line adds something to the page.

st.title adds a title. st.write adds text. st.line_chart draws a chart. st.text_input creates an input box. The code reads like a description of the page. You do not design a layout and then fill it with data. You just write what you want, top to bottom, and Streamlit renders it.

Storyboard -- Streamlit · Code Reads Like English
MEDIA TEAM: Split screen. Left: code being typed line by line. Right: dashboard building in real time. Each st.* call triggers a new UI element to appear on the right. Smooth slide-in animations.
PART B State and Caching
Voiceover + Code ~90 sec

What makes Streamlit powerful for AI is its session state and caching. Session state remembers user inputs across reruns. Caching prevents expensive API calls from running every time the page refreshes. Together, they make it possible to build responsive AI dashboards.

Storyboard -- Caching: 5 Calls → 1 Call
PART C Deployment
Voiceover + Screen ~60 sec

Streamlit also gives you sidebars, columns, tabs, and metric cards for building professional dashboards. And when you are ready to share, Streamlit Community Cloud lets you deploy for free. Push your code to GitHub, connect the repo to Streamlit Cloud, and your app is live on the internet with a public URL.

Shot Setup

Show Streamlit Community Cloud deployment flow: GitHub push, connect repo, app goes live. Highlight the public URL.
